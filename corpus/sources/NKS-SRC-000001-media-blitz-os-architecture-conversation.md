## Summary

This source captures the transition from a Medium/social publishing strategy into a broader Nelson Knowledge System. The key architectural realization is that the corpus does not pre-exist as a folder or dataset; it must be manufactured from source material through a structured knowledge extraction and canonicalization process.

## Durable Ideas Identified

- Media Blitz OS is not the full system; it is the publishing subsystem.
- The broader system should be named or understood as the Nelson Knowledge System.
- The corpus is an output, not an input.
- Source material and canonical corpus must be separated.
- GitHub should function as the control plane and fallback state store.
- Google Drive should remain the editorial workspace when available.
- The No Idle State Rule prevents execution decay.
- Corpus construction requires artifact specifications, source records, and metadata discipline.
