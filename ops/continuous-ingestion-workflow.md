# Continuous Ingestion Workflow

## Purpose

Defines how the Nelson Knowledge System continuously manufactures reusable knowledge from source material without defaulting to publication.

## Core Principle

Ingestion is not storage. Ingestion is manufacturing.

## Workflow

```text
Source Intake
  ↓
Source Classification
  ↓
Durable Idea Extraction
  ↓
Canonical Artifact Creation
  ↓
Proof Boundary Assignment
  ↓
Coherence Scoring
  ↓
Knowledge Graph Linking
  ↓
Opportunity Evaluation
  ↓
Capability Roadmap Update
  ↓
Next Action Selection
```

## Intake Sources

- Conversations
- Documents
- Whitepapers
- Executive profiles
- Resume evidence
- Role analyses
- Diagrams
- Feedback
- Research notes
- Public reactions
- Future system telemetry

## Gate 1: Source Classification

Every source must be classified by:

- source type,
- strategic relevance,
- privacy sensitivity,
- proof potential,
- artifact potential,
- opportunity potential,
- required handling restrictions.

## Gate 2: Durable Idea Extraction

Extract only ideas that can become reusable infrastructure.

Reject:

- temporary commentary,
- emotional noise without durable signal,
- unsupported claims,
- platform-specific wording with no underlying idea,
- duplicates unless they strengthen proof or relationship density.

## Gate 3: Canonical Artifact Creation

Every artifact receives:

- stable ID,
- title,
- source lineage,
- proof boundary,
- capability area,
- relationship map,
- opportunity set,
- next action.

## Gate 4: Coherence Scoring

Apply `templates/coherence-scorecard-template.md`.

Artifacts scoring below 50 remain parked as source material until enriched.

## Gate 5: Graph Linking

Apply `architecture/knowledge-graph-model.md`.

Every canonical artifact must be linked to source, proof, capability, and at least one possible downstream use.

## Gate 6: Opportunity Evaluation

Apply `templates/opportunity-record-template.md`.

Rank opportunities by capability growth and reuse multiplication before publication potential.

## Gate 7: Next Action Selection

Apply `ops/autonomous-prioritization-policy.md`.

The next action is chosen by capability growth, coherence, proof, reuse, automation leverage, and timing.

## Completion Definition

An ingestion item is complete when:

1. Source is classified.
2. Durable ideas are extracted or explicitly rejected.
3. Canonical artifacts are created where justified.
4. Proof boundaries are recorded.
5. Coherence scores exist.
6. Graph links exist.
7. Opportunities are evaluated.
8. Next action is assigned.

## Current Operating Rule

Do not ingest everything.

Ingest what compounds.
