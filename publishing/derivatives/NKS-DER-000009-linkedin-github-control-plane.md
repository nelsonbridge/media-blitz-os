## Core Post Draft

A chat thread is not a control plane.

A folder is not a control plane either.

Those surfaces can hold work, but they do not govern the work.

A serious knowledge system needs somewhere durable to preserve:

- state,
- rules,
- schemas,
- templates,
- sync status,
- and execution history.

That is why GitHub can become useful outside traditional software development.

In this architecture, Drive is the editorial workspace.

GitHub is the control plane.

Drive contains the work.
GitHub contains the machine that governs the work.

That separation creates resilience.

If one surface becomes unavailable, the system can keep moving.
