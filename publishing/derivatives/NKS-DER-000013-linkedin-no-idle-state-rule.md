## Core Post Draft

Waiting is not governance.

In execution systems, useful work should not stop just because one small task finished.

If the next task is known, execute it.

If it succeeds, verify it.

If verification succeeds, update state.

If state is updated, move to the next unblocked task.

That is the No Idle State Rule.

It does not mean the system never stops.

It means the system stops only for real reasons:

- sprint completion,
- genuine blocker,
- unavailable dependency,
- required user decision,
- or public approval boundary.

Everything else should be worked around.

The rule converts approval into momentum.
