## Thread Draft

1. Waiting is not governance.

2. If useful work remains and the next task is known, the system should not become idle.

3. Execute the task. Verify the result. Update state. Continue.

4. Stop only for real reasons: sprint completion, genuine blocker, missing dependency, required user decision, or public approval boundary.

5. Everything else should be worked around.

6. This is the No Idle State Rule.

7. It converts approval into momentum.

8. A system without this rule keeps becoming a conversation again.
