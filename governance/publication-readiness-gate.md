# Publication Readiness Gate

## Purpose

Defines the required gate sequence before any Nelson Knowledge System asset may move toward public release.

## Required Gate Sequence

Source
→ Proof
→ Narrative Arc
→ Visual Package
→ Editorial Review
→ User Approval
→ Publication
→ Derivatives

## Gate 1 — Source

A publication candidate must identify its source lineage.

Required:
- Source Record ID
- Parent Artifact ID
- Source type
- Source status
- Known limitations

## Gate 2 — Proof

A publication candidate must define its proof posture.

Required:
- Proof category
- Evidence boundary
- Citation need
- Unsupported claims to avoid

## Gate 3 — Narrative Arc

A publication candidate must follow the approved reader journey:

Recognition
→ Reframe
→ Framework
→ Proof
→ Application
→ Consequence
→ Invitation

## Gate 4 — Visual Package

A flagship publication must include a Visual Package before public release.

Required:
- Visual Package ID
- Signature Diagram brief
- Hero brief or explicit waiver
- Platform visual requirements
- Proof and narrative boundary for visuals

Visuals are not decorations. They are visual knowledge artifacts and must be governed as part of the corpus/publication package.

## Gate 5 — Editorial Review

The draft must be reviewed for:
- clarity
- factual integrity
- proof alignment
- narrative arc
- brand fit
- visual consistency
- publication risk

## Gate 6 — User Approval

No asset is public-ready until user approval is recorded.

## Gate 7 — Publication

Only after approval may the asset move to Medium or other external publication platforms.

## Gate 8 — Derivatives

Derivatives may be drafted earlier, but public derivatives inherit the same proof, narrative, visual, and approval boundaries as the canonical publication.

## Current Rule

Source first. Proof second. Narrative arc third. Visual package fourth. Publication fifth.
