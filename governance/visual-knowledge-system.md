# Visual Knowledge System

## Purpose

The Visual Knowledge System is the visual manufacturing layer of the Nelson Knowledge System. It exists because the system is no longer only producing written content. It is manufacturing reusable intellectual property from source material, proof, canonical artifacts, narrative structure, and visual evidence.

Visuals are not decorative add-ons. They are first-class knowledge artifacts that explain, prove, compress, and distribute the core architecture of an idea.

## Why This Exists

The Nelson Knowledge System began as a publishing and media strategy, then evolved into a knowledge manufacturing system. That evolution created a missing layer: every canonical artifact needs a visual expression that can be used across Medium, LinkedIn, X, Instagram, Pinterest, presentations, whitepapers, and executive conversations.

Executive audiences often understand systems faster through architecture diagrams, capability maps, operating models, and proof graphics than through prose alone. Therefore, visual artifacts must be governed with the same rigor as written artifacts.

## Governing Principle

No flagship publication is complete until it has a verified Signature Diagram and an aligned Visual Package.

Source

→ Narrative Arc
→ Visual Knowledge Package
→ Publication Package
→ Distribution
→ Feedback

## Visual Pyramid

### Level 1 — Proof Graphics

Purpose: support claims.

Examples:
- ROI charts
- before/after comparisons
- transformation timelines
- operating-metric graphics
- evidence maps

Question answered: How do we know?

### Level 2 — Executive Frameworks

Purpose: explain systems.

Examples:
- operating models
- governance models
- decision trees
- capability maps
- value-chain diagrams
- system architectures

Question answered: How does it work?

### Level 3 — Concept Illustrations

Purpose: make an idea legible.

Examples:
- trust
- compounding
- ambiguity
- transformation
- AI governance
- executive influence

Question answered: What does this mean?

### Level 4 — Attention Assets

Purpose: distribution and recognition.

Examples:
- Medium hero images
- LinkedIn covers
- social quote cards
- Instagram carousels
- Pinterest verticals

Question answered: Will the right reader stop and engage?

5. Social Quote Card
6. LinkedIn Carousel
7. Instagram Carousel
8. Pinterest Vertical
9. Presentation Slide
10. Drive Sync Target

## Visual Integrity Rules

1. Visuals follow source and proof; they do not invent unsupported claims.
2. Visuals must align with the article's narrative arc.
3. Diagrams must teach, not merely decorate.
4. Visual labels must reuse the canonical terminology of the corpus artifact.
5. Every visual artifact receives a stable ID.
6. Every visual has a source lineage and publication relationship.
7. Visual derivatives may not outrun proof, arc, or review gates.

## Relationship to Other NKS Subsystems

- Corpus provides the canonical idea.
- Proof system provides evidence boundaries.
- Narrative Arc Standard provides reader progression.
- Visual Knowledge System provides visual structure.
- Publication Engine packages the written and visual derivatives.
- Drive Sync Ledger tracks editorial workspace synchronization.

## Current Status

The Visual Knowledge System is now a required subsystem of the Nelson Knowledge System. Existing publication candidates must receive visual packages before public release.
