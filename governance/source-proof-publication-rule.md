# Source → Proof → Publication Rule

## Purpose

This rule defines the correct operating sequence for corpus construction and public release.

The Nelson Knowledge System must not treat publication as the next step after drafting. The correct sequence is:

```text
Source
  ↓
Proof
  ↓
Draft
  ↓
Review
  ↓
Publication
  ↓
Derivatives
```

## Rule

No public-facing artifact should move toward publication until its source lineage and proof posture are explicit.

## Source Standard

Every artifact must identify where it came from before it becomes canonical or publication-ready.

Valid source lineage includes:

- Conversation source record.
- Uploaded document.
- Google Drive document.
- GitHub file.
- Email thread.
- Public source.
- Direct observation clearly labeled as experiential.

## Proof Standard

Every artifact must identify what kind of proof supports it.

Proof categories:

- Experiential: based on professional observation or lived operating experience.
- Document-derived: based on uploaded or internal documents.
- Repository-derived: based on implemented GitHub or Drive artifacts.
- Publicly verified: supported by current external sources.
- Quantitative: supported by verifiable metrics or calculations.
- Unverified: not ready for public claim strength.

## Publication Gate

A publication can become public only when:

1. Source lineage is recorded.
2. Proof category is recorded.
3. Claim strength matches proof strength.
4. Required citations or internal evidence references are identified.
5. Brand and career risk are reviewed.
6. User approval is recorded.

## Derivative Rule

Derivatives inherit proof posture from the source publication. A derivative cannot make stronger claims than the source artifact supports.

## Correction to Previous Workflow

Prior GitHub execution correctly created drafts and derivatives, but the forward operating sequence must now prioritize proof before public-readiness review.

Existing drafts remain useful internal assets, but none should be treated as publishable until proof review is complete.

## Operating Principle

Source first. Proof second. Publication third.
