# Diagram Language Standard

## Purpose

Creates a shared visual grammar for Nelson Knowledge System diagrams.

The goal is recognizability, clarity, and executive usability. Diagrams should look like they come from the same operating architecture even when they represent different topics.

## Visual Tone

- Executive report, not influencer graphic.
- Boardroom clarity, not decorative complexity.
- Systems language, not stock illustration.
- High contrast, strong whitespace, minimal noise.

## Shape Language

| Element | Meaning |
|---|---|
| Rectangle | Capability or asset |
| Rounded rectangle | Process or operating step |
| Diamond | Decision point |
| Circle | External actor or stakeholder |
| Hexagon | AI agent or automated actor |
| Shield | Governance, control, or guardrail |
| Cylinder | Data, evidence, or repository |
| Double-bordered box | Canonical artifact |
| Dashed box | Future / queued / unverified state |
| Solid arrow | Direct flow |
| Double arrow | Feedback loop |
| Dashed arrow | Influence or dependency |
| Heavy arrow | Compounding transition |

## Color Semantics

| Color Role | Meaning |
|---|---|
| White | Background / open field |
| Graphite | Primary text / structure |
| Navy | Strategy / governance |
| Steel | Systems / infrastructure |
| Emerald | verified / value / compounding |
| Amber | warning / pending proof |
| Crimson | risk / blocker / failure mode |

## Typography

- Use concise labels.
- Prefer nouns over sentences inside diagram nodes.
- Avoid decorative fonts.
- Use hierarchy: title, subtitle, labels, notes.

## Layout Rules

1. One main idea per diagram.
2. Avoid more than seven primary nodes unless the diagram is a map.
3. Preserve left-to-right or top-to-bottom reading order.
4. Feedback loops must be visually distinct from forward flow.
5. Proof graphics must include source or proof reference marker.
6. Diagrams intended for social platforms must remain legible on mobile.

## Signature Diagram Standard

A Signature Diagram should:

1. Represent the core idea of a flagship publication.
2. Be reusable in Medium, LinkedIn, Slide, and PDF formats.
3. Use canonical terms from the corpus artifact.
4. Avoid unsupported visual claims.
5. Be memorable enough to become associated with the concept.

## Prohibited Patterns

- Random AI art without explanatory value.
- Overly literal stock imagery.
- Dense unreadable architecture maps for social platforms.
- Unverified metrics in graphics.
- Visual metaphors that conflict with the article's proof posture.

## Relationship to Narrative Arc

Each visual should map to at least one arc segment:

- Recognition: make the problem visible.
- Reframe: show the old frame failing.
- Framework: display the new model.
- Proof: show evidence.
- Application: show how to use it.
- Consequence: show what changes.
- Invitation: show the next action.
