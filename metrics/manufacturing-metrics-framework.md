# Manufacturing Metrics Framework

## Purpose

Measure the operating performance of the Nelson Knowledge System without confusing throughput with value.

## Core Principle

Manufacturing metrics must measure capability growth, not content volume.

## Primary Metrics

| Metric | Definition | Why It Matters |
|---|---|---|
| Source-to-Artifact Conversion Rate | Percentage of classified sources that become canonical artifacts. | Measures extraction effectiveness. |
| Artifact Reuse Multiplier | Number of downstream uses per canonical artifact. | Measures leverage. |
| Proof Coverage Rate | Percentage of artifacts with explicit proof boundaries. | Measures claim safety. |
| Graph Link Density | Average meaningful graph edges per artifact. | Measures corpus coherence. |
| Opportunity Generation Rate | Opportunities produced per artifact or cluster. | Measures downstream optionality. |
| Capability Growth Rate | Change in capability maturity over time. | Measures platform evolution. |
| Time to Canonicalization | Time from source intake to stable artifact. | Measures manufacturing speed. |
| Validation Pass Rate | Percentage of artifacts passing required checks. | Measures quality control. |
| Orphan Rate | Percentage of artifacts without source, graph, or opportunity links. | Measures archive drift risk. |
| Trajectory Integrity Score | Degree to which work supports capability roadmap. | Measures strategic alignment. |

## Metric Classes

### Capability Metrics

- Capability maturity delta
- Capability gap closure rate
- New reusable frameworks created
- Validation coverage increase

### Corpus Metrics

- Artifacts created
- Artifacts linked
- Proof-bounded artifacts
- Orphans detected and resolved
- Relationship density

### Opportunity Metrics

- Opportunity portfolios created
- P0 opportunities executed
- Reuse surfaces per artifact
- Output diversity per cluster

### Release-Test Metrics

- Release candidate proof readiness
- Narrative readiness
- Visual readiness
- Feedback capture readiness
- Trajectory risk rating

## Anti-Metrics

Do not optimize for:

- number of posts,
- number of articles,
- number of files,
- number of commits,
- derivative volume,
- platform-specific output frequency.

These may be observed, but they are not success metrics.

## Release-Test Readiness Metric

A release candidate is safe to test when:

1. It has source lineage.
2. It has proof boundary.
3. It has coherence score.
4. It has graph links.
5. It has opportunity rationale.
6. It does not distort the capability roadmap.
7. It has a feedback capture path.
